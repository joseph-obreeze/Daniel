# 판매분석 솔루션 (Sales Analyzer App)

오브리즈(및 브랜드) 판매 데이터를 업로드하여 카테고리별 매출, 옵션별 현황, 월별 트렌드를 분석하는 로컬 데스크톱 어플리케이션입니다.

## 🚀 버전 기록 (Changelog)

### v1.1.3-v5 (현재 버전) - 렌더링 안정성 강화 및 그룹 로직 최적화
* **치명적 렌더링 오류(백화 현상) 완벽 해결**:
  * **원인**: 특정 엣지 케이스에서 상품이나 카테고리의 `revenue`, `qty` 값이 생략(undefined)되었을 때 차트 라벨링을 위해 `toLocaleString()` 등 문자열 변환 함수를 호출하다가 React가 렌더링을 중단하고 전체 화면을 Unmount시키는 현상이 있었습니다.
  * **해결**: 카테고리 분석(`ProductView`)과 옵션별 현황(`SkuView`), 그리고 대시보드 내의 **모든 숫자 포맷팅 영역(20여 곳)**에 `Number(value || 0).toLocaleString()` 형식의 이중 방어막을 설치하여, 비정상 데이터가 유입되더라도 어플리케이션이 뻗지 않도록 수정했습니다.
* **A, B, C, D 상품 그룹화 로직 정교화**:
  * 전체 요약 상위 70% 구간 설정 시, **"딱 70%를 넘어서는 그 제품까지"** 1구간(핵심 그룹)으로 포함하도록 로직(`prevCumulative < 70` 사용)을 개선하여 데이터 누락이나 중복이 없도록 최적화했습니다.

### 이전 주요 업데이트 (v1.1.x)
* **매출 및 각인 매출 분리**: 사용자의 요청에 따라 결제 금액 데이터에서 '각인 서비스' 수익을 완전히 분리해내어, 순수 제품 매출액(`productRevenue`)과 각인 매출액(`engravingTotalRevenue`)을 따로 집계하도록 `cleanData` 파이프라인을 재설계했습니다.
* **엑셀 추출 고도화**: 멀티 시트 기반 엑셀 추출 기능(카테고리별 제품 요약 + 옵션 세부 데이터)을 도입하였고, 전문적인 보고서 디자인(셀 배경색, 볼드 처리, 숫자 포맷팅)을 적용했습니다.

## 💻 실행 및 빌드 방법
* 로컬 테스트 실행: `npm run dev`
* 최종 배포용 빌드(.exe 생성): `npm run electron:build`

---
> **유지보수 주의사항 (Troubleshooting Guide)**
> 향후 데이터 파이프라인(`cleanData`, `calculateSummary`)을 수정할 때는, 새로 생성한 속성(Property) 값이 UI 컴포넌트(`App.jsx` 하단)의 차트 `Tooltip`, `LabelList`, 테이블 등에서 참조될 때 빈 값(undefined, null)이 섞여 들어오지 않도록 각별히 유의해야 합니다.
